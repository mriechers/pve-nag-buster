# PVE 9 modernization — design

**Date:** 2026-07-22
**Fork:** `mriechers/pve-nag-buster` (staging ground for homelab refinements; upstream `foundObjects/pve-nag-buster` stays pristine)
**Target host:** `proxmox-01` — Proxmox VE **9.1.1**, `proxmox-widget-toolkit 5.1.2`

## Problem

Upstream v04 (2021, tested against `pve-manager/6.4-4`) is stale for PVE 9:

1. **Nag patch is brittle and semantically wrong on 9.x.** It does a literal string swap:
   `sed -i.orig "s/data.status.toLowerCase() !== 'active'/false/g"`.
   On 9.1 the real variable is `res.data.status`, so the pattern matches only as a lucky
   substring and rewrites `res.data.status.toLowerCase() !== 'active'` into `res.false`
   — broken JS that happens to be falsy. It works by accident and rots on every refactor.
2. **Repo half is a silent no-op on 9.x.** v04 disables `pve-enterprise.list`, but PVE 9
   ships repos in **deb822 `.sources`** format (`pve-enterprise.sources`, `ceph.sources`,
   `proxmox.sources`). On `proxmox-01` these enterprise repos already carry `Enabled: false`
   (9.1 installer default), so there is nothing for this tool to fix by default.

## Goals / non-goals

- **Goal:** reliably neutralize the login subscription nag on PVE 9.1.1, robustly enough to
  survive cosmetic point-release churn, while keeping upstream's dpkg-hook persistence and
  offline-blob architecture intact.
- **Goal:** keep repo manipulation available but **opt-in**, since host apt config is owned
  by the `proxmox-config` repo.
- **Non-goal:** replacing the tool's architecture, minified-JS support, or supporting PVE < 8.

## Approach decisions

- **Nag patch strategy: robust fail-safe function-match** (free-pmx philosophy), chosen over
  the brittle token swap and over a version-tolerant `data.status` sed.
- **Scope: nag-only by default, repos behind a `--repos` opt-in flag.**

## Design

### 1. Preserved architecture (unchanged)

- `install.sh` flags: `--install` (default) / `--offline` / `--uninstall` / `--emit`.
- Persistence: `/etc/apt/apt.conf.d/86pve-nags` dpkg Pre/Post hooks run
  `/usr/share/pve-nag-buster.sh` (`install -o root -m 0550`) after any apt operation touching
  `pve-manager` / `proxmox-widget-toolkit`.
- Offline base64/xz copy of the hook embedded in `install.sh`, regenerated by
  `make-release.sh`. The vetting invariant **embedded blob == plaintext hook** must hold.

### 2. Nag patch — robust, fail-safe (replaces the string swap)

Neutralize the `checked_command` success handler so the "No valid subscription" modal is never
constructed. On 9.1.1 the target block (proxmoxlib.js lines 632–652) is:

```js
if (
    res === null ||
    res === undefined ||
    !res ||
    res.data.status.toLowerCase() !== 'active'
) {
    Ext.Msg.show({ title: gettext('No valid subscription'), ... });
} else {
    orig_cmd();
}
```

Patch = a single all-or-nothing multi-line Perl substitution replacing that whole conditional
with `orig_cmd();`, using **flexible whitespace** (`\s*` / `\s+`) between tokens so reformatting
across point releases doesn't break the match:

```sh
perl -0777 -i -pe 's/if\s*\(\s*res === null\s*\|\|\s*res === undefined\s*\|\|\s*!res\s*\|\|\s*res\.data\.status\.toLowerCase\(\)\s*!==\s*.active.\s*\)\s*\{.*?\}\s*else\s*\{\s*orig_cmd\(\);\s*\}/orig_cmd();/s' "$NAGFILE"
```

Properties (all verified against the captured 9.1.1 fixture):

- **Correct & isolated:** exactly one substitution; the diff touches only that 21-line block →
  `success: function (response, opts) { let res = response.result; orig_cmd(); }`. The lookalike
  `res.data.status.toLowerCase() !== 'active'` at line 20788 (a Repositories-panel
  `subscriptionActive` status flag, *not* a nag) is deliberately **left untouched** — the
  structural `if (…) { Ext.Msg.show(…) } else { orig_cmd(); }` shape only matches the modal.
- **Fail-safe:** if the block does not match (future refactor), nothing is written and the hook
  exits 0 — the nag stays, the host is otherwise untouched. No partial edits.
- **Backup:** copy `proxmoxlib.js` → `proxmoxlib.js.orig` before the first patch.
- **Restart-on-change only:** compare before/after; run `systemctl restart pveproxy.service`
  only when a substitution actually happened (avoids needless UI blips on every apt run).
- **Idempotent:** after patching, the pattern no longer matches → re-runs are no-ops (verified).

### 3. Repo handling — opt-in, deb822-aware

- **Default install = nag only.** No repo files created, disabled, or moved.
- **`install.sh --repos`** writes marker `/etc/default/pve-nag-buster` containing
  `MANAGE_REPOS=1`. The single hook script sources this marker (if present) and only then:
  - sets `Enabled: false` in `pve-enterprise.sources` and enterprise `ceph.sources`
    (idempotent; also handles legacy `pve-enterprise.list` → `.disabled` on older hosts);
  - ensures a `pve-no-subscription` deb822 source exists (skip if `proxmox.sources` already
    provides `pve-no-subscription`, as on `proxmox-01`).
- One canonical hook script (no per-install variants) keeps the offline blob single-source.

### 4. Uninstall (`--uninstall`)

Remove `/etc/apt/apt.conf.d/86pve-nags`, `/usr/share/pve-nag-buster.sh`, and
`/etc/default/pve-nag-buster`; restore `proxmoxlib.js.orig` if present (then restart pveproxy);
leave repo `.sources` as-is with a printed note (mirrors v04's conservative behavior).

### 5. Testing (no host writes)

Harness in `test/` runs on this Mac against the captured fixture `test/fixtures/proxmoxlib.9.1.1.js`:

- **patch correctness** — after patch, nag block == `orig_cmd();`, rest byte-identical, only the
  expected lines changed;
- **isolation** — the line-20788 status flag is unchanged;
- **fail-safe** — a fixture with the block removed yields zero changes and exit 0;
- **idempotency** — patching twice changes nothing on the second pass;
- **blob invariant** — `install.sh --emit | sha256` == plaintext `pve-nag-buster.sh` sha256.

## Files changed

| File | Change |
|---|---|
| `pve-nag-buster.sh` | Replace token-swap nag logic with the Perl function-match; add `.orig` backup, restart-on-change, fail-safe guard; add optional deb822 repo block gated by `/etc/default/pve-nag-buster`. |

| `README.md` | Note PVE 9 support, robust patch, and `--repos`. |

## Rollout / verification on host

1. Ship + `make-release.sh` re-pack; confirm blob==plaintext.
2. `./install.sh` on `proxmox-01` (nag-only). Confirm nag gone after a browser hard-refresh;
   confirm `proxmoxlib.js.orig` exists.
3. Trigger persistence: `apt reinstall proxmox-widget-toolkit` → hook re-patches automatically.
4. Because host changes touch `proxmox-01`, route the actual on-host run through the
   `proxmox-guardian` per homelab convention.
